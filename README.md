## 🧭 What this app does

AI Daily Digest helps you get a daily AI news brief in WeChat. It gathers AI news from common public sources, builds a short digest, and sends it to your WeChat account on a schedule.

You do not need to keep a server running. You do not need to set up cloud tools. You only need a Windows PC to download the release files and start the app.

## ✨ What you get

- Daily AI news in one place
- WeChat delivery
- No server to manage
- Simple setup on Windows
- Free use with a small monthly cost for message delivery
- A schedule that runs on its own after setup

## 🔑 Set up WeChat delivery

To send the digest to WeChat, the app needs a send key from the message service used by the project.

Follow these steps:

1. Open the message service page used by the project.
2. Sign in or create an account.
3. Create a new send key.
4. Copy the key.
5. Paste the key into the app when asked.

Keep the send key private. It gives the app permission to send messages to your WeChat account.

## ⏰ Choose your daily schedule

Pick a time that fits your day. Common choices are:

- 8:00 AM for a morning brief
- 12:00 PM for a lunch update
- 6:00 PM for an end-of-day recap

The app sends one digest each day at the time you set. If your PC is off, the task will not run, so keep the computer on when the schedule should trigger.

## ❓ If the app does not start

Try these steps:

1. Check that the file finished downloading.
2. Make sure you extracted the ZIP file.
3. Run the app again as an administrator.
4. Check that Windows did not block the file.
5. Make sure your antivirus did not move the file.
6. Download the release again if the file looks damaged.

If the app opens and closes right away, it may need the right files in the same folder.

## 🔔 If messages do not arrive in WeChat

Check these items:

1. The send key is correct.
2. Your internet connection is on.
3. The schedule time has passed.
4. The app is still running.
5. The message service account is active.
6. The test message works before the daily run.

If the test message fails, fix the send key first.
